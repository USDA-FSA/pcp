Controlling column widths

Rather than styling specific widths (fixed or percentage values), use <colspan> to dictate column proportions. While <colspan> is primarily known for combining table cells, it's also a perfect tool for this purpose. The most common reason for using this technique is when you want to align columns across multiple tables.

It's actually a very rarely known method, and I've only ever seen it at Taming data tables. The below is an edited excerpt of that article.

The short version of this section is basically: we are setting up a grid system for our tables.

Taming data tables' columns

Think about page layouts that adhere to a grid system; you might have a 24 column grid, but your page might only have two main columns which span, say, 16 and 8 columns respectively (adding up to 24). You can’t see the 24 columns, but they’re there. You might then have a large footer broken into three columns of eight (again, adding up to 24).

We need to apply this model to tables; we shall give all tables 24 columns, and then use a generous amount of colspans to knock our cells through into each other, into more useful layouts. Now every table we build will be based on a 24 column grid which will, firstly, make everything more consistent, and, secondly, it will make our maths much simpler. We just need to make sure every row’s colspan values add up to 24 every time.

This does mean that every cell in the table now has to carry a colspan, but as I said, this solution is a pragmatic one.

The reason we pick 24 is because it can take halves, thirds, quarters, sixths, eighths and twelfths; we can make a lot of layouts if we have 24 columns to play with.

Now, we would write this snippet:

 ...
 <tr>
   <th>Column one</th>
   <th>Column two</th>
   <th>Column three</th>
 </tr>
 <tr>
   <td>A</td>
   <td>B</td>
   <td>C</td>
 </tr>
 ... 
 ...
 <tr>
   <th colspan="8">Column one</th>
   <th colspan="8">Column two</th>
   <th colspan="8">Column three</th>
 </tr>
 <tr>
   <td colspan="8">A</td>
   <td colspan="8">B</td>
   <td colspan="8">C</td>
 </tr>
 ...

Which with class="fsa-table pcp-table--fixed" renders 3 perfectly distributed columns...
